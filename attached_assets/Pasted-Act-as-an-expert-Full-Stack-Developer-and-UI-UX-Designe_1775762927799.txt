Act as an expert Full-Stack Developer and UI/UX Designer. Build a complete web application for the “Smart Shuttle Solution” for the “42 Irbid Hackathon”.

GOAL
Create a smart booking system where students reserve rides for tomorrow, trips only run when there is enough demand, and both students and drivers can track the trip status in real time.

PROJECT REQUIREMENTS
The application must include:
- Authentication with roles: Student, Admin, Driver
- Student booking page to reserve a ride for tomorrow
- Time slot selection from available trip times
- Pickup point selection from predefined locations only
- Booking status: pending, confirmed, canceled
- Auto-confirmation when enough students book the same trip
- Seat counter with live updates
- Cancel booking feature for students
- Student booking history
- Admin panel to view all bookings
- Driver dashboard with upcoming pickups and passenger counts
- Interactive map with route tracking
- ETA logic for pickup locations
- Real-time updates without refresh
- Mobile-first responsive design

PICKUP POINTS
Use these predefined pickup points:
- Northern Bus Terminal
- Sheikh Khalil Bus Terminal
- Amman Bus Terminal

Students may select only one of these locations. Also, if a student is on or very near the bus route, allow the pickup only when it follows the same booking rules, the point is valid on the route, and the request passes route validation and timing checks.

BONUS FEATURES
Implement these bonus features:
- Interactive Map using Leaflet.js
- Notifications when a trip becomes confirmed or when the bus is approaching a pickup point

DESIGN DIRECTION
Completely overhaul the visual identity to match the modern “42 Intranet V2” aesthetic.

The UI must feel like a premium modern cyber-academy:
- Backgrounds: deep space navy / dark slate blue, not pure black
- Card surfaces: slightly lighter navy with low opacity
- Borders: subtle thin borders, around 1px solid rgba(255,255,255,0.08)
- Accents: neon pink / magenta and cyan / light blue
- Text: pure white for headings, muted grey for secondary text
- Typography: modern sans-serif for main UI such as Inter or Roboto
- Use monospace only for technical data like time slots, IDs, and seat counts
- Rounded corners: 8px to 12px on cards, inputs, and buttons
- Icons: thin-line modern icons such as Lucide or Feather
- Use glowing segment-style progress bars for seat counters, inspired by the 42 intranet progression bar
- Use modern data tables with subtle hover effects
- Add smooth transitions, soft glow effects, and a polished premium feel

PALETTE

- Start mobile-first

- Cards should stack vertically on small screens
- Tables should convert into compact card lists on mobile
- The map should stay usable and full-width on mobile
- Driver UI must be highly legible and safe to read quickly while driving

TECH STACK
Use a modern stack optimized for real-time booking and geolocation:
- Frontend: Next.js + React
- Styling: Tailwind CSS
- Backend: Node.js with Express or NestJS
- Database: PostgreSQL
- Real-time: WebSockets / Socket.IO
- Geolocation and maps: Leaflet.js
- Authentication: JWT or secure session-based auth
- Notifications: Web Push or in-app realtime notifications

REAL-TIME AND GEOLOCATION LOGIC
Design the system so that:
- Seat counts update instantly
- Booking confirmation updates instantly
- Driver and admin dashboards update live
- Bus location is updated in real time
- Pickup points are validated against the predefined route
- Off-route pickup requests are rejected with a clear error
- ETA is calculated for each valid pickup point
- A student must confirm a pickup request at least 15 minutes before the ETA for that stop
- The system shows alerts when the bus is approaching a student’s location
